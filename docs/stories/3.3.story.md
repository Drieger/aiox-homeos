## Story

**As a** usuário do HomeOS,
**I want** editar documentos em Markdown com visualização renderizada,
**so that** eu possa escrever conteúdo formatado de forma elegante.

## Complexity

**Estimativa:** M (Medium) — editor Markdown, toggle preview, auto-save debounce, syntax highlight, tema dark/light.

## Scope

**IN:**
- Integrar `@uiw/react-md-editor` na rota `/kb/[id]` (instalado em Story 3.1)
- Toggle "Editar / Preview" entre editor raw e visualização `react-markdown` + `remark-gfm`
- Classes `prose` do `@tailwindcss/typography` na visualização renderizada
- Título editável inline (campo de texto livre no topo)
- Auto-save título + conteúdo via Server Action com debounce de 1.000ms
- Indicador visual "salvando..." / "salvo" durante o auto-save
- Syntax highlight em blocos de código no preview (`rehype-highlight`)
- Editor e preview respondem ao tema dark/light via `next-themes`
- Testes de comportamento do editor via Playwright (padrão ADR-5/6 para editors complexos)

**OUT:**
- Organização por cadernos/tags (Story 3.4)
- Wiki-links (Story 3.5)
- Exportação (Story 3.7)
- Toolbar de formatação rica (além do que o `@uiw/react-md-editor` fornece nativamente)

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `@uiw/react-md-editor` incompatível com React 19 / Next.js SSR | Média | Alto | Usar `dynamic(() => import(...), { ssr: false })` — editor é Client-only |
| Classes `prose` do typography conflitar com tema dark | Baixa | Médio | Usar `prose dark:prose-invert` para inversão automática |
| Auto-save criar race condition com múltiplos debounce | Baixa | Médio | Usar `useRef` para cleanup do timeout; cancelar debounce pendente ao desmontar |
| `rehype-highlight` aumentar bundle size | Baixa | Baixo | Import lazy; apenas no preview (não no editor) |

## Acceptance Criteria

1. Editor Markdown `@uiw/react-md-editor` integrado na rota `/kb/[id]` — exibe conteúdo existente ao abrir
2. Toggle "Editar / Preview" alterna entre editor raw e visualização renderizada com `react-markdown` + `remark-gfm`
3. Visualização renderizada aplica classes `prose dark:prose-invert` do `@tailwindcss/typography` para tipografia elegante
4. Título do documento é editável inline (campo `<input>` ou `<textarea>` no topo da página)
5. Auto-save salva título + conteúdo automaticamente com debounce de 1.000ms após última modificação
6. Indicador visual exibe "salvando..." durante o debounce e "salvo" após confirmação da Server Action
7. Syntax highlight em blocos de código no preview via `rehype-highlight`
8. Editor responde ao tema dark/light via `next-themes` (sem FOUC)

## Dev Notes

### Nota arquitetural crítica (ADR-5/6)

> O editor `@uiw/react-md-editor` usa CodeMirror internamente. Testes de comportamento de editors complexos devem usar **Playwright** (browser real), não Vitest/JSDOM. [Source: docs/prd/epic-3-knowledge-base.md — Notas Arquiteturais]

### Import dinâmico obrigatório

```typescript
// "use client" obrigatório
import dynamic from "next/dynamic";
const MDEditor = dynamic(() => import("@uiw/react-md-editor"), { ssr: false });
```

O `@uiw/react-md-editor` usa APIs de browser (CodeMirror) e não funciona em SSR.

### Hook de auto-save

```typescript
// src/hooks/use-auto-save.ts
"use client";
import { useRef, useState, useEffect } from "react";

export function useAutoSave(
  id: string,
  title: string,
  content: string,
  saveFn: (id: string, data: { title: string; content: string }) => Promise<void>
) {
  const [status, setStatus] = useState<"idle" | "saving" | "saved">("idle");
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  useEffect(() => {
    if (timerRef.current) clearTimeout(timerRef.current);
    timerRef.current = setTimeout(async () => {
      setStatus("saving");
      await saveFn(id, { title, content });
      setStatus("saved");
      setTimeout(() => setStatus("idle"), 2000);
    }, 1000);
    return () => { if (timerRef.current) clearTimeout(timerRef.current); };
  }, [id, title, content]);

  return status;
}
```

### Tema dark/light no editor

```typescript
import { useTheme } from "next-themes";

const { resolvedTheme } = useTheme();
```

```tsx
<div className="prose dark:prose-invert max-w-none p-4">

    {content}
  </ReactMarkdown>
```

### Server Action — getDocumentById

```typescript
// src/lib/kb/actions.ts
export async function getDocumentById(id: string) {
  const result = await db.select().from(documents).where(eq(documents.id, id)).limit(1);
  return result[0] ?? null;
}
```
