## Story

**As a** usuário do HomeOS,
**I want** buscar documentos por qualquer texto no título ou conteúdo,
**so that** eu possa encontrar informações rapidamente sem saber onde estão.

## Complexity

**Estimativa:** M (Medium) — overlay de busca global, FTS5 query, debounce, highlight de trechos.

## Scope

**IN:**
- Atalho `Ctrl+K` (Windows/Linux) abre overlay de busca global
- Campo de busca consulta FTS5 em tempo real (debounce 300ms)
- Resultados exibem: título, caderno, trecho do conteúdo com termo destacado
- Clicar em resultado navega para `/kb/[id]` e fecha overlay
- Performance: < 200ms para base com até 10.000 documentos
- Botão de busca visível na UI do KB (header ou sidebar) como alternativa ao atalho
- Mensagem "Nenhum resultado encontrado" para busca sem resultados
- Testes de integração para a Server Action FTS5 e testes E2E para o overlay

**OUT:**
- Busca global fora do módulo KB (v2+)
- Filtros avançados por caderno/tag no overlay de busca (Story 3.4 cobre filtros na sidebar)
- Histórico de buscas (v2+)
- Busca por expressão regular (v2+)

2. Campo de busca consulta FTS5 em tempo real com debounce de 300ms

## Dev Notes

### FTS5 query — SQL raw (padrão confirmado de Story 3.1)

O Drizzle ORM não suporta FTS5 nativamente. Usar `db.$client` (instância `better-sqlite3`, confirmada em `src/lib/db/index.ts`) para queries FTS5.

**Padrão da FTS5 (confirmado em Story 3.1):** `content=documents, content_rowid=rowid` — tabela externa. O `JOIN` com `documents` é feito via `rowid`.

```typescript
export async function searchDocuments(query: string) {
  if (!query.trim()) return [];
  // Sanitizar e adicionar * para prefix search
  const ftsQuery = query.trim().replace(/[^a-zA-Z0-9\s]/g, "") + "*";

  // snippet(fts_table, iCol, start, end, ellipsis, nTokens)
  // iCol: 0 = title, 1 = content (ordem das colunas definida em initKbFts: title, content)
  const stmt = db.$client.prepare(`
    SELECT d.id, d.title, n.name as notebook_name,
           snippet(documents_fts, 1, '<mark>', '</mark>', '...', 10) as snippet
    FROM documents_fts
    JOIN documents d ON d.rowid = documents_fts.rowid
    JOIN notebooks n ON n.id = d.notebook_id
    WHERE documents_fts MATCH ?
    ORDER BY rank
    LIMIT 20
  `);

  return stmt.all(ftsQuery) as SearchResult[];
}
```

**Atenção:** `snippet()` retorna HTML com `<mark>` — usar `dangerouslySetInnerHTML` apenas para o snippet, não para o título.

**iCol = 1** corresponde à coluna `content` (índice 0 = `title`, índice 1 = `content`) conforme definição da FTS5 em `initKbFts()`:
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS documents_fts USING fts5(
  title,    -- iCol 0
  content,  -- iCol 1
  content=documents, content_rowid=rowid
);
```

### Sanitização da query FTS5

FTS5 tem sintaxe própria — caracteres especiais como `"`, `(`, `)`, `*`, `^` podem causar erros de parse. Remover caracteres não-alfanuméricos antes de passar para o MATCH.

### Hook de atalho de teclado

```typescript
// src/hooks/use-kb-search.ts
"use client";
import { useEffect } from "react";
import { create } from "zustand";

const useSearchStore = create<{ isOpen: boolean; toggle: () => void; close: () => void }>(
  (set) => ({
    isOpen: false,
    toggle: () => set((s) => ({ isOpen: !s.isOpen })),
    close: () => set({ isOpen: false }),
  })
);

export function useKbSearch() {
  const { isOpen, toggle, close } = useSearchStore();
  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if (e.ctrlKey && e.key === "k") {
        e.preventDefault();
        toggle();
      }
    };
    document.addEventListener("keydown", handler);
    return () => document.removeEventListener("keydown", handler);
  }, [toggle]);
  return { isOpen, close };
}
```
