## Story

**As a** desenvolvedor,
**I want** um projeto Next.js 15 configurado com TypeScript strict, Tailwind CSS, shadcn/ui e ferramentas de qualidade,
**so that** todas as stories subsequentes tenham uma base consistente e sem configuração duplicada.

## Complexity

**Estimativa:** M (Medium) — 10 tasks de configuração inicial envolvendo múltiplas ferramentas independentes com dependências de versão.

## Scope

**IN:**
- Inicializar projeto Next.js 15 com App Router e TypeScript
- Configurar Tailwind CSS v4, shadcn/ui, ESLint, Prettier
- Instalar e configurar Vitest + Testing Library + Playwright
- Criar estrutura de diretórios conforme source-tree.md
- Escrever 1 teste unitário de validação do setup (TDD)

**OUT:**
- Implementação de qualquer feature de produto
- Criação de componentes UI além do `button` (validação do shadcn/ui)
- Configuração de banco de dados (Story 1.2)
- Qualquer estilização ou layout (Stories 1.3, 1.4)

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Incompatibilidade Tailwind v4 + shadcn/ui | Média | Alto | Verificar release notes do shadcn/ui para suporte ao Tailwind v4; ajustar config se necessário |
| `create-next-app` gera tsconfig incompatível com strict extras | Baixa | Médio | Task 2 cobre explicitamente as flags adicionais |
| Playwright conflita com versão Node.js instalada | Baixa | Baixo | Verificar compatibilidade antes de executar `npm init playwright@latest` |

## Acceptance Criteria

1. Next.js 15+ criado com App Router e TypeScript strict mode habilitado
2. Tailwind CSS v4 configurado com `@import "tailwindcss"` (sem `tailwind.config.js`)
3. shadcn/ui inicializado com `components.json` apontando para `src/components/ui/`
4. ESLint + Prettier configurados com regras padrão do projeto
5. Vitest + Testing Library instalados com configuração base em `vitest.config.ts`
6. Playwright instalado com configuração base em `playwright.config.ts`
7. `npm run dev`, `npm run build`, `npm run test` e `npm run lint` executam sem erros
8. Estrutura de diretórios criada: `src/app/`, `src/components/`, `src/lib/`, `src/hooks/`, `src/store/`, `src/lib/queries/`, `src/lib/commands/`
9. Pelo menos 1 teste unitário escrito (TDD: test-first) validando a configuração do Vitest

## Dev Notes

### Stack e Versões

- **Next.js**: 15+ com App Router — `npx create-next-app@latest`
- **TypeScript**: 5+ strict mode — ver configuração completa em `docs/architecture/coding-standards.md#1-typescript---configuração-strict`
- **Tailwind CSS**: v4 — usa `@import "tailwindcss"` no CSS global, **SEM** `tailwind.config.js`
- **shadcn/ui**: CLI `npx shadcn@latest init` — componentes vão para `src/components/ui/`
- **Vitest**: substitui Jest — mais rápido para ESM, compatível com Next.js
- **Playwright**: para E2E e testes do TipTap (ProseMirror não funciona em JSDOM)

import { defineConfig } from "vitest/config";
import react from "@vitejs/plugin-react";
import { resolve } from "path";

export default defineConfig({
  plugins: [react()],
  test: {
    environment: "jsdom",
    setupFiles: ["./src/test-setup.ts"],
    globals: true,
  },
  resolve: {
    alias: {
      "@": resolve(__dirname, "./src"),
    },
  },
});

### Padrão TDD para esta story

O teste de validação do Vitest (AC 9) deve ser escrito ANTES do setup estar completo, seguindo Red → Green:

```typescript
// tests/unit/setup.test.ts
import { describe, it, expect } from "vitest";

describe("Vitest setup", () => {
  it("ambiente de teste está funcional", () => {
    expect(true).toBe(true);
  });

  it("matchers do jest-dom estão disponíveis", () => {
    const element = document.createElement("div");
    document.body.appendChild(element);
    expect(element).toBeInTheDocument();
  });
});
```

### Testing

- **Localização dos testes unitários**: `tests/unit/`
- **Localização dos testes de componente**: `tests/component/`
- **Localização dos testes E2E**: `tests/e2e/`
- **Framework unit/integration/component**: Vitest + `@testing-library/react`
- **Framework E2E**: Playwright
- **Padrão**: TDD obrigatório — testes escritos ANTES da implementação
- **Environment**: `jsdom` para testes com DOM; nenhum environment para testes puramente unitários
- **Nota sobre TipTap** (relevante para stories futuras): ProseMirror não funciona em JSDOM — qualquer teste que envolva o editor deve usar Playwright
