## Story

**As a** desenvolvedor,
**I want** uma página `/design-system` estruturada com seções e navegação interna,
**so that** os componentes possam ser adicionados de forma organizada ao longo do projeto.

## Complexity

**Estimativa:** S (Small) — página estática com layout e 4 seções placeholder, sem dados ou interatividade.

## Scope

**IN:**
- Rota `/design-system` em `src/app/(app)/design-system/page.tsx` (substituir placeholder)
- 4 seções com id de âncora: "Tipografia", "Cores", "Componentes de Formulário", "Overlays"
- Layout 2 colunas desktop: nav lateral fixo + área de conteúdo principal
- Scroll suave via CSS `scroll-behavior: smooth`
- Sidebar exibe "Design System" como item ativo na rota
- 1 teste de componente (TDD) validando as 4 seções

**OUT:**
- Conteúdo real de tipografia/cores (Story 2.2)
- Showcases de form elements (Story 2.3)
- Showcases de overlays (Story 2.4)
- Qualquer componente interativo além de links de navegação

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `scroll-behavior: smooth` não aplicado globalmente | Baixa | Baixo | Adicionar ao CSS global ou inline na página |
| Sidebar active state não detecta `/design-system` | Baixa | Médio | Verificar implementação de `app-sidebar.tsx` antes de codificar |

## Acceptance Criteria

1. Rota `/design-system` criada em `src/app/(app)/design-system/page.tsx`
2. Página renderiza seções navegáveis: "Tipografia", "Cores", "Componentes de Formulário", "Overlays"
3. Navegação interna via âncoras com scroll suave
4. Layout da página usa grid de 2 colunas em desktop: menu lateral fixo + área de conteúdo
5. Sidebar do app exibe "Design System" como item ativo ao navegar para esta rota
6. Teste de componente (TDD): validar que as 4 seções são renderizadas
