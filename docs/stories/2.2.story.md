## Story

**As a** usuário,
**I want** visualizar a escala tipográfica e a paleta de cores do HomeOS,
**so that** eu tenha uma referência visual dos tokens de design do sistema.

## Complexity

**Estimativa:** S-M (Small-Medium) — componentes de display estático, mas requer acesso ao sistema de temas (useTheme) e mapeamento correto dos CSS variables do shadcn/ui.

## Scope

**IN:**
- Seção "Tipografia" com H1, H2, H3, H4, body, small, muted, code — com nome do token e exemplo
- Seção "Cores" com tokens CSS do tema shadcn/ui (light + dark side-by-side)
- Cada swatch: nome do token, CSS variable (`--background`, etc.) e preview colorido
- Ambas as seções respondem ao toggle de tema global
- 1 teste de componente (TDD)

**OUT:**
- Editor de tokens ou customização de cores
- Exportação de tokens
- Animações ou transições além do theme switch nativo

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Lista de CSS variables do shadcn/ui incompleta | Média | Baixo | Mapear as variáveis diretamente do `globals.css` gerado pelo shadcn/ui init |
| `useTheme` causa hydration mismatch | Baixa | Médio | Usar `mounted` state para renderizar swatches apenas client-side |

3. Cada swatch exibe: nome do token, valor CSS variable e preview colorido
4. Ambas as seções respondem ao toggle de tema

### Cores — CSS variables do shadcn/ui

O shadcn/ui define variáveis em `globals.css` (gerado pelo `npx shadcn@latest init`) no formato `--background: oklch(...)` ou `hsl(...)`. Para mostrar light + dark simultaneamente, renderizar dois grupos com classes `.light` / `.dark` explícitas ou usar `data-theme` attributes. [tech-stack.md — next-themes]

### Client Component obrigatório

`useTheme` do `next-themes` requer `"use client"`. O componente de cores deve ser Client Component. Componente de tipografia pode ser Server Component (sem estado). [coding-standards.md#2-padrões-de-componentes-react]

### Hydration SSR-safe

```typescript
const [mounted, setMounted] = useState(false)
useEffect(() => setMounted(true), [])
if (!mounted) return null // evita mismatch
```

[tech-stack.md — next-themes SSR-safe com `suppressHydrationWarning`]

**Único concern bloqueante (MEDIUM):** O grupo "Light" em `ColorsSection` aplica a classe CSS `.light` ao container, porém `globals.css` define apenas `:root` e `.dark` — não existe seletor `.light`.
