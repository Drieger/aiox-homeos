## Story

**As a** usuário do HomeOS,
**I want** organizar meus documentos em cadernos e tags,
**so that** eu possa encontrar conteúdo relacionado facilmente.

## Complexity

**Estimativa:** M (Medium) — seletor de caderno, CRUD de cadernos, tags com autocomplete, filtros na sidebar.

## Scope

**IN:**
- Seletor de caderno no editor (`/kb/[id]`) para mover documento entre cadernos existentes
- Botão "Novo Caderno" na sidebar KB com input inline de nome
- Campo de tags no editor: autocomplete com tags existentes + criação de novas
- Tags exibidas como chips removíveis no editor
- Filtro por caderno na sidebar KB (clique → filtra lista de documentos)
- Filtro por tag na sidebar KB (clique → filtra lista)
- Opção "Todos os documentos" limpa filtros
- Server Actions para gerenciar cadernos e tags
- Testes de integração para as Server Actions de organização

**OUT:**
- Renomear ou excluir cadernos/tags (v2+)
- Hierarquia de cadernos (sub-cadernos) — fora do escopo v1
- Mover múltiplos documentos simultaneamente (v2+)
- Ordenação customizada de documentos (v2+)

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Autocomplete de tags causar excesso de queries | Baixa | Baixo | Buscar todas as tags uma vez no mount; filtrar localmente |
| Filtro de caderno/tag quebrar URL routing | Baixa | Médio | Usar Zustand store (não URL params) para estado de filtro — simplifica implementação |
| Race condition entre seleção de caderno e auto-save | Baixa | Baixo | Seletor de caderno usa `updateDocument` independente do debounce de conteúdo |

## Acceptance Criteria

1. Seletor de caderno no editor (`/kb/[id]`) permite mover o documento para qualquer caderno existente via `<Select>` do shadcn/ui
2. Botão "Novo Caderno" na sidebar cria caderno com nome editável (input inline) e persiste ao pressionar Enter ou ao perder foco
3. Campo de tags no editor permite adicionar tags existentes via autocomplete (dropdown) ou criar novas digitando e pressionando Enter
4. Tags são exibidas como chips removíveis (badge com botão `×`) no editor
5. Clicar em um caderno na sidebar filtra a lista de documentos para mostrar apenas documentos daquele caderno
6. Clicar em uma tag na sidebar filtra a lista para documentos com aquela tag
7. Opção "Todos os documentos" na sidebar limpa os filtros ativos e mostra todos os documentos

## Dev Notes

### Zustand store para estado de filtro

```typescript
// src/stores/kb-store.ts
import { create } from "zustand";

interface KbState {
  activeNotebookId: string | null;
  activeTagId: string | null;
  setActiveNotebook: (id: string | null) => void;
  setActiveTag: (id: string | null) => void;
  clearFilters: () => void;
}

export const useKbStore = create<KbState>((set) => ({
  activeNotebookId: null,
  activeTagId: null,
  setActiveNotebook: (id) => set({ activeNotebookId: id, activeTagId: null }),
  setActiveTag: (id) => set({ activeTagId: id, activeNotebookId: null }),
  clearFilters: () => set({ activeNotebookId: null, activeTagId: null }),
}));
```

### Filtro por tag — SQL com JOIN

```typescript
// Ao filtrar por tagId em getDocuments:
const result = await db
  .select({ document: documents })
  .from(documents)
  .innerJoin(documentTags, eq(documentTags.documentId, documents.id))
  .where(eq(documentTags.tagId, tagId))
  .orderBy(desc(documents.updatedAt));
```

### addTagToDocument — idempotente

```typescript
export async function addTagToDocument(documentId: string, tagName: string) {
  // Criar tag se não existe
  let tag = await db.select().from(tags).where(eq(tags.name, tagName)).limit(1);
  if (tag.length === 0) {
    const id = ulid();
    await db.insert(tags).values({ id, name: tagName });
    tag = [{ id, name: tagName }];
  }
  // Vincular (ignorar se já existe)
  await db.insert(documentTags).values({
    documentId,
    tagId: tag[0].id,
  }).onConflictDoNothing();
  revalidatePath(`/kb/${documentId}`);
}
```

### Padrão de timestamps

Todos os updates devem incluir `updatedAt: Date.now()`. O `moveDocument` deve atualizar `updatedAt` do documento para refletir a reorganização.
