## Story

**As a** usuário,
**I want** alternar entre tema claro e escuro com minha preferência salva,
**so that** o sistema respeite minha escolha visual entre sessões.

## Complexity

**Estimativa:** S (Small) — Providers simples, store Zustand com expansão do existente, ThemeToggle com shadcn/ui. Depende de 1.1 e 1.3.

## Scope

**IN:**
- `next-themes` + `ThemeProvider` no root layout
- Expandir `src/store/ui-store.ts` com campo `theme` (sem recriar)
- `src/components/layout/theme-toggle.tsx` com dropdown 3 opções
- `src/components/providers.tsx` compondo ThemeProvider + QueryClientProvider
- Atualizar `src/app/layout.tsx` com `<Providers>` e `suppressHydrationWarning`
- Canary page com nome "HomeOS", ThemeToggle e link para `/design-system`
- Testes unitários TDD para o store Zustand (mutations de tema e sidebar)

**OUT:**
- Design System completo (Epic 2)
- Preferências de usuário persistidas em banco de dados
- Internacionalização (i18n) ou múltiplos idiomas
- Sistema de notificações ou toasts globais
- Qualquer lógica de autenticação

### `src/store/ui-store.ts` — Store Completo

```typescript
import { create } from "zustand";
import { persist } from "zustand/middleware";

type Theme = "light" | "dark" | "system";

interface UIStore {
  sidebarCollapsed: boolean;
  theme: Theme;
  setSidebarCollapsed: (collapsed: boolean) => void;
  toggleSidebar: () => void;
  setTheme: (theme: Theme) => void;
}

export const useUIStore = create<UIStore>()(
  persist(
    (set) => ({
      sidebarCollapsed: false,
      theme: "system",
      setSidebarCollapsed: (collapsed) => set({ sidebarCollapsed: collapsed }),
      toggleSidebar: () =>
        set((state) => ({ sidebarCollapsed: !state.sidebarCollapsed })),
      setTheme: (theme) => set({ theme }),
    }),
    { name: "ui-store" }
  )
);
```

**Nota:** Se a Story 1.3 criou uma versão mínima deste store, esta story o expande — não recria do zero. Verificar se o arquivo já existe.

### `src/components/providers.tsx`

```typescript
"use client";

import { ThemeProvider } from "next-themes";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { useState } from "react";

export function Providers({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient());

  return (
    <ThemeProvider
      attribute="class"
      defaultTheme="system"
      enableSystem
      disableTransitionOnChange
    >
      <QueryClientProvider client={queryClient}>
        {children}
      </QueryClientProvider>
    </ThemeProvider>
  );
}
```

**Importante:** `QueryClient` deve ser criado com `useState(() => new QueryClient())` — não fora do componente — para evitar compartilhamento de state entre requisições em SSR.

[Source: docs/architecture/source-tree.md#componentes-providers]

```typescript
import type { Metadata } from "next";
import { Inter } from "next/font/google";
import { Providers } from "@/components/providers";
import "./globals.css";

const inter = Inter({ subsets: ["latin"] });

export const metadata: Metadata = {
  title: "HomeOS",
  description: "Seu sistema pessoal de produtividade",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="pt-BR" suppressHydrationWarning>
      <body className={inter.className}>
        <Providers>{children}</Providers>
      </body>
    </html>
  );
}
```

**`suppressHydrationWarning` é obrigatório** no `<html>` quando usando next-themes — o provider modifica o atributo `class` no lado do cliente e o React emite warning sem isso.

### `src/components/layout/theme-toggle.tsx`

```typescript
"use client";

import { useTheme } from "next-themes";
import { Moon, Sun, Monitor } from "lucide-react";
import { Button } from "@/components/ui/button";
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from "@/components/ui/dropdown-menu";

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button variant="ghost" size="icon">
          <Sun className="h-4 w-4 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
          <Moon className="absolute h-4 w-4 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
        </Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent align="end">
        <DropdownMenuItem onClick={() => setTheme("light")}>
          <Sun className="mr-2 h-4 w-4" /> Claro
        </DropdownMenuItem>
        <DropdownMenuItem onClick={() => setTheme("dark")}>
          <Moon className="mr-2 h-4 w-4" /> Escuro
        </DropdownMenuItem>
        <DropdownMenuItem onClick={() => setTheme("system")}>
          <Monitor className="mr-2 h-4 w-4" /> Sistema
        </DropdownMenuItem>
      </DropdownMenuContent>
    </DropdownMenu>
  );
}
```

### Testing

- **Localização**: `tests/unit/store/ui-store.test.ts`
- **Framework**: Vitest (sem DOM — store é lógica pura)
- **Mock necessário**: `localStorage` (Zustand `persist` usa localStorage)
- **Padrão TDD**: testes escritos ANTES do store

```typescript
import { describe, it, expect, beforeEach } from "vitest";
import { useUIStore } from "@/store/ui-store";

// Resetar store antes de cada teste
beforeEach(() => {
  useUIStore.setState({
    sidebarCollapsed: false,
    theme: "system",
  });
});

describe("ui-store", () => {
  it("sidebarCollapsed inicia como false", () => {
    expect(useUIStore.getState().sidebarCollapsed).toBe(false);
  });

  it("toggleSidebar alterna o estado", () => {
    useUIStore.getState().toggleSidebar();
    expect(useUIStore.getState().sidebarCollapsed).toBe(true);
    useUIStore.getState().toggleSidebar();
    expect(useUIStore.getState().sidebarCollapsed).toBe(false);
  });

  it("setTheme define o tema corretamente", () => {
    useUIStore.getState().setTheme("dark");
    expect(useUIStore.getState().theme).toBe("dark");
  });
});
```
