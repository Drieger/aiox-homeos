## Story

**As a** usuário do HomeOS,
**I want** criar links entre documentos com `[[nome]]` e ver quais documentos me linkam,
**so that** eu possa construir uma rede de conhecimento interconectada.

## Complexity

**Estimativa:** H (High) — popover de busca no editor, custom renderer para Markdown, extração de links, painel de backlinks, tabela `document_links`.

## Scope

**IN:**
- Digitar `[[` no editor abre Popover de busca de documentos existentes
- Selecionar documento no popover insere `[[Título do Documento]]` no texto
- Custom renderer `react-markdown`: `[[Título]]` vira link clicável
- Clicar no link no preview navega para `/kb/[id]` do documento
- Ao salvar: extrair todos os `[[links]]` e popular tabela `document_links`
- Painel "Backlinks" lateral: lista documentos que linkam o atual
- Links para documentos inexistentes: estilo `text-muted-foreground` no preview

**OUT:**
- Grafo visual de conexões (Obsidian-style) — fora do escopo v1
- Preview de hover no wiki-link (v2+)
- Links para headings dentro de documentos (v2+)
- Aliases nos wiki-links `[[Título|Alias]]` (v2+)

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Popover de wiki-links no editor conflitar com CodeMirror | Alta | Alto | Detectar `[[` via `onChange` do conteúdo; exibir Popover fora do editor, posicionado relativamente |
| Custom renderer quebrar outros elementos Markdown | Baixa | Médio | Aplicar renderer apenas para o node de texto; usar regex apenas em `[[...]]` patterns |
| Extração de links no save ser lenta com muitos links | Baixa | Baixo | Regex simples; operação local síncrona antes da Server Action |
| `document_links` com links órfãos após renomear doc | Média | Baixo | Fora do escopo MVP; aceitar links quebrados como comportamento esperado |

## Acceptance Criteria

1. No editor, digitar `[[` abre um Popover de busca de documentos existentes (lista filtrável por título)
2. Selecionar um documento no Popover insere `[[Título do Documento]]` no cursor e fecha o Popover
3. No preview renderizado, `[[Título do Documento]]` aparece como link clicável (estilo `text-primary underline`)
4. Clicar no link no preview navega para o documento destino (`/kb/[id]`)
5. Ao salvar o documento, o sistema extrai todos os `[[links]]` do conteúdo e atualiza a tabela `document_links` (remove links obsoletos, insere novos)
6. Painel "Backlinks" lateral exibe lista de documentos que linkam o documento atual com título e link clicável
7. Links para documentos inexistentes (`[[Título Inexistente]]`) são exibidos em `text-muted-foreground` sem link no preview

## Dev Notes

### Detecção de `[[` no editor — abordagem

O editor `@uiw/react-md-editor` expõe o conteúdo como string via `onChange`. Para detectar um wiki-link em progresso:

```typescript
useEffect(() => {
  // Detecta "[[" sem fechar "]]" no final do conteúdo
  const match = content.match(/\[\[([^\]]*?)$/);
  if (match) {
    setWikiQuery(match[1]); // texto após "[["
    setShowWikiPopover(true);
  } else {
    setShowWikiPopover(false);
    setWikiQuery("");
  }
}, [content]);
```

### Custom renderer para ReactMarkdown

```tsx
// src/lib/kb/wiki-link-renderer.tsx
// Pré-processar o conteúdo antes de passar para ReactMarkdown:
// Substituir [[Título]] por um placeholder HTML-safe, depois usar component customizado

// Alternativa mais simples: processar o texto como string antes do render
function processWikiLinks(text: string, docMap: Map<string, string>): string {
  return text.replace(/\[\[([^\]]+)\]\]/g, (_, title) => {
    const id = docMap.get(title);
    if (id) return `[${title}](/kb/${id})`;
    return `<span class="muted-wiki">[[${title}]]</span>`;
  });
}
```

**Nota:** Esta abordagem de pré-processamento é mais simples que criar um rehype plugin. Transforma `[[Título]]` em Markdown link padrão antes de passar para ReactMarkdown.

### syncDocumentLinks — lógica de sincronização

```typescript
export async function syncDocumentLinks(sourceId: string, content: string) {
  // 1. Extrair todos os [[titles]] do conteúdo
  const matches = [...content.matchAll(/\[\[([^\]]+)\]\]/g)].map(m => m[1]);

  // 2. Resolver títulos para IDs
  const resolvedIds: string[] = [];
  for (const title of matches) {
    const doc = await db.select({ id: documents.id })
      .from(documents)
      .where(eq(documents.title, title))
      .limit(1);
    if (doc.length > 0) resolvedIds.push(doc[0].id);
  }

  // 3. Substituir links (delete source + insert novos)
  await db.delete(documentLinks).where(eq(documentLinks.sourceId, sourceId));
  if (resolvedIds.length > 0) {
    await db.insert(documentLinks).values(
      resolvedIds.map(targetId => ({ sourceId, targetId }))
    ).onConflictDoNothing();
  }
}
```
