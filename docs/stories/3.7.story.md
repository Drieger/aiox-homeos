## Story

**As a** usuário do HomeOS,
**I want** exportar documentos e ter uma experiência refinada de uso,
**so that** o módulo KB seja completo e prazeroso de usar no dia a dia.

## Complexity

**Estimativa:** S (Small) — exportação via download, responsividade, persistência de estado, navegação com teclado.

## Scope

**IN:**
- Botão "Exportar .md" na toolbar do editor que baixa conteúdo como `[titulo].md`
- Arquivo exportado contém apenas Markdown puro (sem metadados)
- Layout responsivo: em telas < 768px, sidebar KB e lista de documentos colapsam
- Estado da sidebar KB (expandida/colapsada) persiste no Zustand store `useKbStore`
- Navegação com teclado no overlay de busca: setas ↑↓, Enter para selecionar, Esc para fechar
- Todos os textos da interface KB em português
- Revisão geral de UX/acessibilidade do módulo KB

**OUT:**
- Exportação para PDF (v2+)
- Upload de imagens (aceito apenas via link externo `![alt](url)`)
- Exportação em lote de múltiplos documentos (v2+)
- Import de arquivos .md externos (v2+)

## Dependencies

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Download de arquivo não funcionar em todos os browsers | Baixa | Baixo | Usar `<a download>` com Blob URL — suportado em todos os browsers modernos |
| Sidebar colapsável quebrar layout 3 colunas | Baixa | Médio | Testar com `transition-width` e `overflow-hidden`; fallback para `display:none` |
| Navegação com teclado no overlay precisar de refactor | Baixa | Médio | Usar `aria-activedescendant` + índice controlado por state; padrão simples |

## Acceptance Criteria

1. Botão "Exportar .md" na toolbar do editor baixa o conteúdo do documento como arquivo `[titulo].md` no browser
2. O arquivo exportado contém apenas Markdown puro — sem front matter, metadados ou JSON
3. Em telas < 768px, sidebar KB e lista de documentos colapsam (padrão `Sheet` ou ocultação do HomeOS)
4. Estado da sidebar KB (expandida/colapsada) persiste via Zustand store (sobrevive a navegação entre documentos)
5. No overlay de busca (Story 3.6): setas ↑↓ movem o foco entre resultados, Enter navega para o selecionado, Esc fecha o overlay
6. Todos os textos da interface do módulo KB estão em português (botões, labels, placeholders, mensagens de estado)

## Dev Notes

### Download de arquivo — padrão

```typescript
function handleExport() {
  const blob = new Blob([content], { type: "text/plain;charset=utf-8" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = `${title.replace(/[^a-z0-9]/gi, "_").toLowerCase()}.md`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

Sem bibliotecas externas — apenas APIs nativas do browser.

### Layout responsivo — estratégia

Em mobile (< 768px), usar `Sheet` do shadcn/ui para a sidebar KB (slide-in lateral). A lista de documentos fica visível na tela inicial `/kb`; ao entrar em `/kb/[id]`, ocultar a lista.


### Navegação com teclado — padrão

```tsx
// Em search-overlay.tsx
const [focusedIndex, setFocusedIndex] = useState(-1);

const handleKeyDown = (e: React.KeyboardEvent) => {
  if (e.key === "ArrowDown") setFocusedIndex(i => Math.min(i + 1, results.length - 1));
  if (e.key === "ArrowUp") setFocusedIndex(i => Math.max(i - 1, 0));
  if (e.key === "Enter" && focusedIndex >= 0) {
    router.push(`/kb/${results[focusedIndex].id}`);
    onClose();
  }
  if (e.key === "Escape") onClose();
};
```

Reset `focusedIndex` para -1 quando `query` muda.
