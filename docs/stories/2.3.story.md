## Dev Notes

### Dependência

Esta story **depende de Story 2.1** (Design System Page Scaffold). A seção "Componentes de Formulário" na rota `/design-system` deve existir antes de implementar o showcase. Implementar após Story 2.1 estar Done.

### Localização dos componentes shadcn/ui

`src/components/ui/` — instalados via `npx shadcn@latest add {componente}`. Import: `import { Button } from "@/components/ui/button"`. [source-tree.md]

### Client Component obrigatório

Select, Checkbox e Switch gerenciam estado interno — o `FormElementsSection` deve ser Client Component com `"use client"`. [coding-standards.md#2-padrões-de-componentes-react]

### Button — variantes e disabled

```typescript
import { Button } from "@/components/ui/button"
// variant="default|destructive|outline|secondary|ghost|link"
// disabled prop para estado desabilitado
```

### Select (shadcn/ui)

```typescript
import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from "@/components/ui/select"
```

### Badge — variantes

```typescript
import { Badge } from "@/components/ui/badge"
// variant="default|secondary|destructive|outline"
```

### Checkbox e Switch com Label

```typescript
import { Checkbox } from "@/components/ui/checkbox"
import { Label } from "@/components/ui/label"

<div className="flex items-center gap-2">
  <Checkbox id="example" />
  <Label htmlFor="example">Exemplo</Label>
</div>
```

### Gotcha: text-foreground em containers de tema (Story 2.2)

Em qualquer container com classe de tema explícita (`.light` / `.dark`), **todos** os elementos de texto precisam de `text-foreground` ou `text-muted-foreground` explícitos — nunca depender da cor padrão do browser. Sem isso, texto preto sobre fundo escuro = invisível.

```typescript
// ✅ Correto
<span className="text-sm font-medium text-foreground">Label</span>

// ❌ Errado — cor padrão do browser (preto) pode ser invisível em dark
<span className="text-sm font-medium">Label</span>
```
